netPhotoGraphics filters

Filters modify the behavior of existing functions and can be used by any of the plugin types. Examples of where a filter is used are noted with each filter. Registered filters handlers are run in priority sequence and can modify the output of earlier handler functions. So, for instance, the seoFriendly() function "applies" the seoFriendly filter passing the text that is to be made "friendly". Each registered handler makes its own changes to the text and the result is the cumulative set of changes. Handlers may also be used to directly create output. The theme_head filter is '"applied" by theme scripts from within their <head>.../<head> tags so that plugins can put CSS or JavaScript content there.

Filters are added to a function (which has filtering support) by using the function npgFilters::register($hook, $function_name, $priority). $hook is the name of the filter item, $function_name is the function that will be invoked to filter the item, and $priority is the rank of this filter among the list of filters for this item (defaults to plugin's priority as set with $plugin_is_filter). Filters are applied in ascending $priority order.

The registered function will be passed the parameters(with the exception of the filter name string) from the npgFilters::apply() call. The function must always return the initial parameter of the list (if it exists) as this is then passed on to subsequent filters in the chain. This provides a means of filters operating on the changes made by other filters.
